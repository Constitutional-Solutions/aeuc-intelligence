# Chapter 13 – Pattern Vocabulary and Design Space

This chapter introduces a shared vocabulary for application patterns.
It helps the family describe new deployments in terms that connect back to Parts I–III.

## 13.1 Pattern building blocks

Every pattern is described in terms of:

- **Domain** – the human space it serves (e.g. accounting, safety engineering, research).
- **Primary axis and sector** – which part of the Love · Truth · Power geometry is in focus.
- **Kernel hooks** – which core services are involved (audit, scheduler, Sabbath, reflection, buddies, registry).
- **Governance flows** – which votes, gates, or playbooks are required.
- **Artifacts** – what inputs and outputs the pattern handles (files, ledgers, models, reports).

## 13.2 Pattern template

A pattern entry uses the following template:

- **Name**
- **Domain and primary axis**
- **Intent** (what this pattern protects or advances)
- **Involved lenses**
- **Kernel services used**
- **Key axioms enforced**
- **Inputs / outputs**
- **Example flow**

Later chapters (14–16) fill this template for specific domains.
